# Phase 1: 项目初始化与基础设施
# 目标：搭建骨架、配置环境、日志系统

## 1.1 目录结构创建
- [ ] 手动或通过脚本创建完整目录树
- [ ] 确保以下空目录存在：
  - `assets/world/`
  - `assets/personas/`
  - `assets/materials/`
  - `data/sessions/`
  - `data/logs/`
  - `src/core/`
  - `src/interface/`
  - `src/clients/`
  - `src/utils/`

## 1.2 全局配置系统
- [ ] 创建文件：`src/utils/config.py`
  - [ ] 定义类：`AppConfig`
    - [ ] 方法：`__init__(self, config_path="config/settings.yaml")`
    - [ ] 逻辑：按优先级加载配置 (命令行 > 环境变量 > YAML文件)
  - [ ] 交互：作为单例模式，被后续所有模块导入使用。

- [ ] 创建文件：`config/settings.yaml` (模板)
  - [ ] 配置项：`llm.api_key`, `llm.base_url`
  - [ ] 配置项：`paths.persona_dir`, `paths.session_dir`
  - [ ] 配置项：`emotion.enabled`, `emotion.default_render_position`
  - [ ] 配置项：`memory.refine_strategy`, `memory.compression_ratio`

## 1.3 日志系统
- [ ] 创建文件：`src/utils/logger.py`
  - [ ] 定义函数：`setup_logger(name, log_file="data/logs/runtime.log")`
  - [ ] 逻辑：同时输出到文件和控制台，区分 DEBUG/INFO/WARNING/ERROR
  - [ ] 预留：专门的 Logger Name 空间，如 `Logger("EMOTION")`, `Logger("MEMORY")`
  - [ ] 交互：被所有核心模块导入使用。

## 1.4 依赖管理
- [ ] 创建文件：`requirements.txt`
  - [ ] 列入：`pyyaml`, `python-dotenv`, `aiohttp` (可选，用于异步)
